Platform Overview

Baitlab is a full-stack AI platform combining voice AI, workflow automation, and real-time 3D visualisation. The platform is built on a microservices architecture with event-driven communication and horizontal scaling.

🧠 AI Core

LLM-powered reasoning engine with custom fine-tuning, RAG pipelines, and tool-use capabilities.

🔄 Orchestration

Visual workflow builder with conditional branching, loops, error handling, and webhook triggers.

📊 Analytics

Real-time dashboards tracking call outcomes, automation success rates, and system health.

System Architecture

Three-tier architecture: API Gateway → Microservices → Data Layer. Event-driven with Apache Kafka for async communication. Kubernetes for container orchestration.

Layer 1

API Gateway

Rate limiting, auth, routing, and load balancing across service replicas.

Layer 2

Microservices

AI Engine, Voice Service, Workflow Engine, Analytics Service — independently deployable.

Layer 3

Data Layer

PostgreSQL for structured data, Redis for caching, S3-compatible object storage, Kafka for streams.

Security & Compliance

Baitlab is built with a security-first mindset: all data in transit is encrypted with TLS 1.3, data at rest uses AES-256, and PII is automatically redacted from logs.

🔐 Encryption

TLS 1.3 in transit. AES-256 at rest. Customer-managed keys available.

📋 Compliance

SOC 2 Type II, GDPR, HIPAA-ready. Audit logs for all actions.

🛡️ Access Control

RBAC with SSO/SAML, MFA enforcement, and session management.

Quick Start Guides

  1. Create a Baitlab account and navigate to AI Agents.
  2. Click New Agent and choose Voice Agent from the templates.
  3. Configure the agent's name, voice, and greeting script.
  4. Set up routing rules (e.g. forward to human after 3 unresolved turns).
  5. Publish and assign a phone number — the agent goes live immediately.
  1. Go to Workflows → New Workflow.
  2. Select a trigger: webhook, schedule, or AI call completion event.
  3. Add action nodes: send email, update CRM, book calendar slot, etc.
  4. Add conditional branches for different outcomes.
  5. Test with sample data and deploy. Monitor runs in the dashboard.
